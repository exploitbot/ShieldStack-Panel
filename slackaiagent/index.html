üí¨ Using the Bot in Slack

Test Your Setup

  1. Send to bot: "What is 2+2?"
  2. Wait 2-3 seconds
  3. Send: "pull 80" (or "show 50")
  4. See the agent's response!

Basic Commands

  • pull - Get last 80 lines from log
  • show - Same as pull but defaults to 50 lines
  • pull 200 / show 200 - Request a custom number of lines (max 400)
  • status - Check system health
  • logpath on/off - Reveal or hide the log directory
  • command help - Sends `/help` inside the agent screen (any text works)
  • help - Show command list

Slash Commands

  • /agent status - Service status
  • /agent restart - Restart bridge
  • /agent stop - Send Ctrl+C
  • /agent recent - Recent output
Note: Any text not recognized as a command is sent directly to the agent for processing.

üèóÔ∏è How It Works

System Flow

  1. You send a message in Slack
  2. Slack bot receives it
  3. Screen bridge injects into the agent session
  4. The agent processes with full capabilities
  5. Output logged to file
  6. You run pull / show to see the response

Components

  • slack_bot.py - Main bot handler
  • screen_agent_bridge.py - Bridge to screen
  • GNU Screen - Session: slack-claude
  • AI CLI (Claude) - With full permissions
  • Log File - logs/claude_screen.log

üîß Quick Troubleshooting

Common Issues

  • Empty pull/show results
    ‚Üí The agent is still processing. Wait a bit and run pull 200 or show 200.
  • Old/stale output
    ‚Üí Run: ./init.sh start
  • Bot not responding
    ‚Üí Check: ./init.sh test
  • Service crashed
    ‚Üí Run: sudo systemctl restart slack-claude-bot

  • screen -ls - List sessions
  • tail -f logs/claude_screen.log - Watch log
