If you see an error when typing /agent in Slack, it's because the slash command isn't registered yet.

This is optional! You can use regular messages instead (see workaround below).

üìù Register /agent Slash Command

Step 1: Go to Slack App Settings

  1. Visit https://api.slack.com/apps
  2. Select your app: EricAIAgents (App ID: A09SW99R5MX)

Step 2: Create Slash Command

  1. In left sidebar, click Slash Commands
  2. Click Create New Command

Step 3: Configure the Command

Fill in these details:

Field Value
Command /agent
Request URL https://example.com/slack/commands
(Not used in Socket Mode, but required by Slack)
Short Description Control and manage the AI agent bot
Usage Hint [status|restart|stop|clear|newsession|help]

Click Save

Step 4: Reinstall App (if prompted)

  1. Go to Install App in left sidebar
  2. If you see "Reinstall" button, click it
  3. Authorize the permissions

‚úÖ Verification

Test the slash command in Slack:

/agent status

Expected response: Agent status: *running*

ü§ñ Available /agent Commands

Command Description
/agent status Check if bot is running
/agent stop NEW! Stop current operation (Ctrl+C)
/agent restart Restart the agent process
/agent clear Clear conversation context
/agent newsession Generate new session ID and restart
/agent help Show all available commands

üí° Workaround: Use Regular Messages

Can't register slash commands? No problem! Use regular messages instead:

Instead of... Type...
/agent help agent help
/agent status agent status
/agent stop agent stop

The bot understands both!
