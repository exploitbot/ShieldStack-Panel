Step 2: Start the agent CLI in screen
    mkdir -p /home/appsforte/slackaiagent/logs

           -S slack-claude \
           -dm bash -lc 'cd /home/appsforte/slackaiagent && /home/appsforte/.nvm/versions/node/v24.10.0/bin/claude --dangerously-skip-permissions'

Step 3: Restart service
    sudo systemctl restart slack-claude-bot
    sudo systemctl status slack-claude-bot

===============================================================================
4. SLACK USAGE PATTERN
===============================================================================

Send any message → it is typed directly into the agent's screen session.
Then run one of:
  • `pull 80`     – tail the log (default from `CLAUDE_SCREEN_PULL_DEFAULT`)
  • `show 50`     – alias of pull with a 50-line default
  • `pull N` / `show N` – request a custom number of lines (max 400)
  • `command help` – send `/help` inside the agent session
  • `command connectvpn` – built-in VPN helpers (`autoconnectvpn`, `stopvpn`)
  • `status`      – check screen/log availability
  • `logpath on|off` – reveal or hide the log directory in Slack replies
  • `help`        – show quick instructions

Wait ~1–2 seconds between sending a command and running `pull`/`show` so the agent finishes responding.

===============================================================================
5. OPTIONAL SLASH COMMANDS (/agent …)
===============================================================================

  /agent status   – bridge running? (does NOT check screen)
  /agent restart  – restarts bridge
  /agent stop     – sends Ctrl+C to bridge
  /agent recent   – dumps PTY buffer
